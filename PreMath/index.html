Precision Math

Why I made this thing: -First started when I learned that 0.1 + 0.2 === 0.3 evaluated to false. When I dug a little deeper, I saw that natively implemented Math and arithmetic had some limitations. Also as someone teaching math using computer, I needed a way to store rational number and calculate using them. -For Steps Math stuff (made in the past for things from past, but improved it after realizing its value) -Just to see if I can -To use it for future projects, such as solving KenKen (link) You can use these in the console.
What is *?
It means the user can pass numbers as strings as opposed to numbers. In other words, instead of passing 12345678901234567890123456789, which gets evaluated to 1.2345678901234568e+28, user can pass an argument as '12345678901234567890123456789'.

PreMath.factorial()

Although native multiplication is fast, when attempting to obtain a factorial of a large number, the user gets a number in scientific notation. PreMath.factorial() returns a value in string without truncation. The first argument is the integer for which the factorial is obtained. The second argument must be set to true if the first integer is larger than 10000. Passing the second argument as true means the user acknowledges that it might take a bit of time for things to be processed.
